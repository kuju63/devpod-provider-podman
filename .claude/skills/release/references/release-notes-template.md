## Formatting Guidelines

1. **Use emojis consistently** across all releases
2. **Keep issue references** as markdown links
3. **Code blocks** use triple backticks with language hint
4. **Links section** always includes comparison link
5. **Footer** always includes Claude Code credit
6. **Line length**: No hard limit, rely on GitHub's markdown rendering
7. **Blank lines**: One blank line between sections

## Quality Checklist

- [ ] Version number is correct
- [ ] All categories have appropriate emojis
- [ ] Issue links are functional
- [ ] Installation instructions are up-to-date
- [ ] Comparison link uses correct previous version
- [ ] Documentation link points to tagged version (not main)
- [ ] Footer credit is present
- [ ] Breaking changes highlighted if present
